# Todo JPA

책 『스프링 부트 개발자 온보딩 가이드』 실습 내용을 기반으로 **Spring Boot 프로젝트를 체계적으로 관리하기 위한 학습용 레포지토리**입니다.  
실습, 개념 정리, 개선 작업을 GitHub 이슈와 칸반 보드를 활용하여 관리합니다.

---

## 📌 프로젝트 목표
- **To-Do List REST API** 앱에 데이터 영속성 지원 기능을 추가합니다.
- **JPA**를 활용하여 도메인 모델과 데이터베이스 간의 매핑 과정을 학습합니다.
- 인메모리 기반 프로젝트에서 **영속 계층**으로 확장되는 구조를 이해합니다.

## 💡요구사항

- 기능 
  - JPA와 MySQL 데이터베이스를 이용하여 CRUD(Create, Read, Update, Delete) 기능을 구현합니다.
- 구현
  - 엔드포인트: 모든 API 엔드포인트는 "/api/todos/v2"로 시작합니다.
  - 테스트: 각 CRUD 기능에 대한 유닛 테스트를 작성합니다.
  - 문서화 및 테스트: Swagger3를 사용하여 API 문서화를 수행하고 테스트를 가능하게 합니다.

## 🗂 이슈 관리 방식
이 프로젝트의 모든 작업은 GitHub 이슈를 통해 관리합니다.

- **실습(Task)**  
  책의 실습 단위를 관리하기 위한 이슈

- **개념 정리(Concept)**  
  학습 중 정리해야 할 개념 기록

- **개선/리팩터링(Refactor)**  
  코드 구조나 설정 개선 작업

이슈는 GitHub Projects(칸반 보드)를 통해  
`To do → In Progress → Done` 흐름으로 관리합니다.
